Step Three: Prototype

Once you had your goals and knew what needed to do, start creating the components.

  • Get to know the tool

    1. Constraints

      Constraints let you "stick" an element to a direction upon resizing.

    2. Auto Layout

      Use the auto layout for things you know would require specific sizing and elements that had lists.

  • Naming Convention

    The next step is to build out the information architecture. To make the design system easy to navigate, its structure had to be easy to understand.

  • Colors

    The final step for this version is to sync up the colors. This is a significant pain point for the engineer. The naming of the color did not match what was in code.

    For this, pull all the names from the code and spent time matching the colors and renaming them. This will make sure, that when the engineer code the design, he can copy the variable name and not have to look up the matching.
