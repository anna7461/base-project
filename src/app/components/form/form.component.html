Usage

Use forms when user input is required to complete some action. Forms direct users to provide and submit relevant information such as changing personal settings or creating a Jira issue.

Common form components include text fields, selects, radio buttons, checkboxes, date pickers, toggles, and buttons.

Anatomy

  1. Title: Use a concise form title. This helps the user understand the form’s context.

  2. Labels: Standard label alignment is left-aligned with the field underneath. Use a combination of a concise label and supportive helper text for clarification of field input.

  3. Placeholder text: Clearly communicate fields that are required. Don’t rely on asterisks only to communicate which fields are required. Don’t use placeholder text to communicate that a field is required.

  4. Required field: Clearly communicate fields that are required. Don’t rely on asterisks only to communicate which fields are required. Don’t use placeholder text to communicate that a field is required.

  5. Form section: Group information logically into form sections.

  6. Disabled fields: Show disabled fields if users need to know what controls might not be available to them.

  7. Form footer: Always use a submit button and distinguish primary and secondary buttons.

Best practices

General

Persist forms on refresh so that user data isn’t lost.

Layout

Group related sections into logical sets to allow users to understand the information they need to enter.

Use one form column to enable fast field comprehension.

Max-width: 624px, recommended by the cozy grid. In other cases, assume that the width typically follows where the form lives in (Modal, focused task, page, etc.)

Fields and labels

  • Reduce the number of fields in your form to avoid a large cognitive load on users (or to help users submit the form faster).

  • Avoid optional fields by only asking users for relevant information.

  • Avoid using default values or instead, use smart defaults (e.g. preselect user's country based on location data).

Use input masks to help users understand correct field formats.

Buttons

  • State the action that the buttons perform when clicked.

  • Avoid multiple action buttons and avoid a reset button. This will reduce confusion about how to submit the form.

Ensure the primary action button provides feedback that the form is being submitted.

Button positioning

Buttons are sorted by importance from right to left, with the most important button (primary) on the right.

Exceptions to this are single-page forms, flags, and section messages where buttons are left aligned with the primary button on the left.

Accessibility

  • Autofocus the first field by default. This allows users to tab through elements in the form in a logical way.

  • Never disable a submit button, even if all of the required fields aren't filled in. Instead, describe what needs to be done using validation and error messages.

  • Match keyboard to input on mobile.

  • Use autocompletion to limit typing for users.
