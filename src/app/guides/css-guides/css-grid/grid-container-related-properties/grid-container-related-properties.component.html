Properties Related to the Grid Container

  1. The grid-template-columns and grid-template-rows properties.

    These 2 properties specify the size of the grid tracks and line names. The property value is expressed as a space-separated list, known as track list.

    The grid-template-columns property defines the track list for all the columns in the grid, while the grid-template-rows property defines the track list for all the rows in the grid. Line names are optional.

    Because grid was made to be very flexible in terms of how developers can define very customized grids, there are many different options available to define the values of these properties.

    • none

      This is the initial value. No explicit grid-tracks are created.

    • < track-list >

      A list of values that specify the size of each grid track, and grid line names, which are optional.

    • < track-size >

      There are 3 ways you can define < track-size >.

      1. < track-breadth >

        • Can be a CSS length or percentage, like 250px or 20%.

        • Can be a flexible length, declared using the fr unit, which takes up a share of the remaining free space in proportion to its flex factor.

          For example: Given a grid container of 750px with 3 columns of 150px, 1 fr and 2fr respectively. The flexibility sized columns will take up the remaining 600px in the ration of 1:2, so the second column takes up 200px and the third column takes up 400px.

        • Can be the keyword min-content, which is the smallest size possible what does not lead to overflow.

        • Can be the keyword max-content, which is the minimum size required to fit around the contents.

        • Can be the keyword auto, which will fill up the remaining space available.

      2. minmax ( < inflexible-breadth >, < track-breadth > )

        • Can be a range as defined by the minmax() function, where the first value is the minimum and the second value is the maximum. For this case, the minimum value cannot be a flexible length, so you can use all the types of values as < track-breadth > except flexible units.

      3. fit-content ( < length-percentage > )

    • < track-repeat >

      As for < track-repeat >, it means we can have repeating < track-size >. We can do that with the repeat() function that lets us specify a large number of columns or rows that follow a similar pattern.

    • < line-names >

      If you're creating a more complicated grid, for example, to layout the entire website, it might be useful to name the grid lines, to make the grid code easier to understand.

      Line names can be any string except "span", which is a keyword.

      Grid lines can have more than one name.

    • < auto-track-list >

      A list of values that specify the size of each grid track, and grid line names, which are optional.

      • < fixed-size >

        There are three ways you can define < fixed-size >.

        1. < fixed-breadth >

          Can be any CSS length or percentage value.

        2. minmax < fixed-breadth >, < track-breadth >

          Can be a range as defined by the minmax() function, where the first value is the minimum and the second value is the maximum. For this case, the minimum value must be a < fixed-breadth >, while the maximum value can be any values allowed for < track-breadth >.

        3. minmax ( < inflexible-breadth >, < fixed-breadth > )

          Can be a range as defined by the minmax() function, where the first value is the minimum and the second value is the maximum. For this case, the minimum value cannot be a flexible unit, while the maximum value must be a < fixed-breadth >.

      • < fixed-repeat >

        By using the repeat() notation, we can create a grid where we can specify how many times a fixed-size grid column or row should be repeated.

      • < auto-repeat >

        By using the repeat() notation, we can create a grid, where a fixed-size grid column or row can be repeated to fill up the available space.

        The auto-fill keyword will generate as many columns as will fit the available space without causing the grid to overflow.

        The auto-fil keyword behaves similarly. The difference is that auto-fit will collapse any empty repeated tracks(which means their size is 0px).

  2. The grid-template-area property.

    This property defines named grid areas and provides a visualization of the grid structure, which may help make underlying code easier to understand. The possible values have the following meanings:

    • none

      This is the initial value. No explicit grid tracks are created, and hence no named grid areas are defined.

    • < string >

      Each separate string creates a row while each word in the string creates a column. All strings must have the same number of words, otherwise the declaration is invalid. Using a sequence of one or more '.' (U+200E FULL STOP) represents a null call token, which is a unnamed area in the grid.

  3. The grid-template property.

    This is a shorthand for setting < grid-template-columns >, < grid-template-rows > and < grid-template-areas > in the single declaration.

    The possible values and syntax forms have the following meanings:

    • none

      This sets all 3 properties to their initial values (which is none).

    • < grid-template-row's > / < grid-template-columns' >

      This sets the < grid-template-rows' > and < grid-template-columns' > to specified values, while < grid-template-areas' > is set to none.

      Note, that the order of the syntax is important, where the first value is form < grid-template-rows' > and the second value is for < grid-template-columns' >.

    • [< line-names > ? < string > < track-size > ? < line-names > ?] + [/explicit-track-list>]?

      This sets the < grid-template-areas' > to the listed strings.

      The <'grid-template-rows'> will be set to the < track-size >s that come after each string. The grid lines will also be named based on the line names sandwiching each string.

      The <'grid-template-columns'> will be set to the track listing specified after the slash. If nothing is specified after the slash, it will be set to none.

  4. The grid-column-gap and grid-row-gap properties.

    These properties specify the gutter size between grid columns and grid rows respectively. Gutters do not appear on the edges of the grid which means there is not gutter before the first and last tracks.

    • < length-percentage >

      Can be any CSS length or a percentage of the parent's container width.

  5. The grid-gap property.

    This is a shorthand for setting <'grid-row-gap'> and <'grid-column-gap'> in a single declaration. The first value sets the <'grid-row-gap'> and the second value sets <'grid-column-gap'>.

    If the <'grid-column-gap'> is not present, it will take the same value as <'grid-row-gap'>.

    The possible values have the following meanings:

    • <'grid-row-gap'>

      Can be any CSS length or a percentage of the parent's container width.

    • <'grid-column-gap'>

      Can be any CSS length or a percentage of the parent's container width.
